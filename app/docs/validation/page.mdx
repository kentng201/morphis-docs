# Validation

Morphis includes a built-in validation engine with no extra dependencies.
Validators are typed, composable, and live alongside your models and controllers.

---

## Generate a validator

Run the generator with a PascalCase name ending in `Validator`:

```bash
morphis new:validator PostBodyValidator
```

This creates `src/validators/PostBodyValidator.ts` with the full scaffold:

```ts filename="src/validators/PostBodyValidator.ts"
import { SimpleValidationRuleMap, ValidationRule, Validator } from 'morphis';

export interface PostBody {
    // TODO: define your data shape here
}

export class PostBodyValidator extends Validator<PostBody> {
    getSimpleRules(): SimpleValidationRuleMap<PostBody> {
        const { Required } = this.rules;
        return {
            // TODO: add per-field rules
            // example: name: [Required],
        };
    }

    getRules(): ValidationRule<PostBody>[] {
        return [
            // TODO: add cross-field rules
        ];
    }
}
```

<Callout type="info">
  Names must be PascalCase and end with `Validator`. Examples: `OrderValidator`, `UserProfileValidator`.
</Callout>

## SimpleRules reference

`SimpleRules` (also available as `this.rules` inside the class) covers the most common cases.

### Presence

| Rule | Behavior |
| ---- | -------- |
| `Required` | Must be present, non-null, non-empty string, non-empty array |
| `Optional` | Skips the field entirely |
| `Nullable` | `null` passes; `undefined` fails |
| `Nullish` | Both `null` and `undefined` pass |

### Format

| Rule | Behavior |
| ---- | -------- |
| `Email` | Must match `user@domain.tld` |
| `Numeric` | Must be a number |
| `Boolean` | Must be `true` or `false` |
| `Date` | Must parse as a valid date |
| `Alphanumeric` | Letters and digits only |
| `Uppercase` | All uppercase |
| `Lowercase` | All lowercase |
| `NoSpecialCharacters` | No special characters |
| `Positive` | Number greater than 0 |
| `Negative` | Number less than 0 |

### Parameterised

| Rule | Example | Behavior |
| ---- | ------- | -------- |
| `Min(n)` | `Min(3)` | Value or length ≥ n |
| `Max(n)` | `Max(100)` | Value or length ≤ n |
| `Length(max, min?)` | `Length(255)` | String character count |
| `Size(max, min?)` | `Size(10)` | Array item count |
| `Between(min, max)` | `Between(1, 5)` | Numeric range |
| `GreaterThan(n)` | `GreaterThan(0)` | Strictly above n |
| `LessThan(n)` | `LessThan(100)` | Strictly below n |
| `Regex(pattern)` | `Regex(/^\d+$/)` | Must match pattern |
| `In(...values)` | `In('draft', 'published')` | Must be one of the values |
| `Enum(enumObj)` | `Enum(Status)` | Must be a TypeScript enum value |
| `Decimals(max, min?)` | `Decimals(2)` | Decimal place count |

## Complete PostBodyValidator

```ts filename="src/validators/PostBodyValidator.ts"
import { SimpleValidationRuleMap, ValidationRule, Validator } from 'morphis';
import { Post } from '../models/Post';

export type PostBody = Partial<Post>;

export class PostBodyValidator extends Validator<PostBody> {
    getSimpleRules(): SimpleValidationRuleMap<PostBody> {
        const { Required, Length, In } = this.rules;
        return {
            title:  [Required, Length(255)],
            body:   [Required, Length(10000)],
            status: [Required, In('draft', 'published', 'archived')],
        };
    }
}
```

Attach it to the controller action with `@Validate`:

```ts
@Post()
@Validate({ body: PostBodyValidator })
async create(req: Request) {
    return await PostModel.create(req.body as Partial<PostModel>);
}
```

Each field holds an array of messages so multiple rules on the same field are reported together.

## strictCheck — clean and formatted output

By default validators are permissive: unknown fields simply pass through to `req.body`.
Set `readonly strictCheck = true` to reject undeclared fields and have the middleware
replace `req.body` with the sanitised output that contains **only the declared keys**.

```ts filename="src/validators/PostBodyValidator.ts"
export class PostBodyValidator extends Validator<PostBody> {
    readonly strictCheck = true;  // ← enable

    getSimpleRules(): SimpleValidationRuleMap<PostBody> {
        const { Required, Length, In } = this.rules;
        return {
            title:  [Required, Length(255)],
            body:   [Required, Length(10000)],
            status: [Required, In('draft', 'published', 'archived')],
        };
    }
}
```

**Request body sent by the client**

```json
{
  "title":           "Hello",
  "body":            "Some content",
  "status":          "draft",
  "extraDescription": "should be ignored"
}
```

**Response — 400**

```json
{
  "errors": {
    "extraDescription": ["extraDescription is not an accepted field"]
  }
}
```

The request is rejected before the controller runs — `req.body` inside `PostController.create`
**never contains `extraDescription`**. This prevents mass-assignment issues without any extra
filtering in the controller.

<Callout type="info">
  Even when the request is rejected, the framework internally strips unknown keys from
  `result.output`. This means downstream code that manually calls `.validate()` always
  receives a clean object regardless of the `valid` flag.
</Callout>

## getRules() — complex validation logic

`getRules()` is for cases that `SimpleRules` cannot express in a single field: cross-field
checks, async database lookups, or nested/array field inspection.

### Object-level rule (no key)

When `key` is omitted the predicate receives the **whole object**.
Use destructuring for readable code:

```ts
getRules(): ValidationRule<PostBody>[] {
    return [
        {
            rule: ({ body }) => !/<script[\s>]/i.test(body ?? ''),
            message: '$body must not contain script tags',
        },
    ];
}
```

The `$body` token in `message` maps the error to the `body` field in the response.

### Key-based rule (field or dot-notation path)

When `key` is provided, the predicate receives the value **at that path** — fully typed by
TypeScript. For nested objects and arrays Morphis traverses the path automatically using
dot-notation, so `'tags.name'` iterates every item in the `tags` array with `name` typed correctly.

```ts
export interface PostBody {
    title:  string;
    body:   string;
    status: string;
    tags?:  { name: string; weight: number }[];
}

export class PostBodyValidator extends Validator<PostBody> {
    readonly strictCheck = true;

    getSimpleRules(): SimpleValidationRuleMap<PostBody> {
        const { Required, Length, In } = this.rules;
        return {
            title:       [Required, Length(255)],
            body:        [Required, Length(10000)],
            status:      [Required, In('draft', 'published', 'archived')],
            'tags.name':   [Required, Length(50)],
            'tags.weight': [Required],
        };
    }

    getRules(): ValidationRule<PostBody>[] {
        return [
            // Object-level: destructure for clean multi-field checks
            {
                rule: ({ title, body }) =>
                    !/<script[\s>]/i.test(title ?? '') &&
                    !/<script[\s>]/i.test(body  ?? ''),
                message: '$title and $body must not contain script tags',
            },
            // Key-based: predicate is typed to `{ name: string; weight: number }`
            {
                key: 'tags',
                rule: ({ weight }) => weight > 0 && weight <= 100,
                message: '$weight must be between 1 and 100',
            },
        ];
    }
}
```

<Callout type="info">
  The `key` field supports full dot-notation (`'tags.name'`, `'address.city'`).
  When the path resolves to an array, the predicate runs once per element and errors
  are reported with index-qualified keys such as `tags.0.weight`.
</Callout>

### Async rules

Predicates can be `async`, which lets you check the database before accepting the request:

```ts
getRules(): ValidationRule<PostBody>[] {
    return [
        {
            key: 'title',
            rule: async (title) => {
                const existing = await PostModel.findOne({ where: { title } });
                return !existing;
            },
            message: '$title is already taken',
        },
    ];
}
```
